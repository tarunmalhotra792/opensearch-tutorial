About OpenSearch


Welcome to the world of OpenSearch, a robust and flexible search engine toolkit designed to help you search, visualize, and analyze your data in real-time.

Originally developed as a fork of Elasticsearch and Kibana, OpenSearch was created by Amazon Web Services (AWS) and is governed by the open-source community.

OpenSearch consists of a search engine (OpenSearch) and a visualization tool (OpenSearch Dashboards), which work together to help organizations build scalable, high-performance search and analytics solutions.


Why OpenSearch?


  • Scalability: Handle large datasets with ease, allowing you to grow without performance bottlenecks.
  • Flexibility: Customize and extend OpenSearch to fit your specific use cases, thanks to its open-source nature.
  • Community-Driven: Benefit from an active and growing community of developers and users who contribute to the continuous improvement of the platform.
  • Real-Time Insights: Leverage powerful features like distributed search, aggregations, and advanced analytics to gain immediate insights from your data.

Key Features


  • Open Source: Entirely free to use, modify, and distribute, OpenSearch is committed to maintaining its open-source roots.
  • Full-Text Search: Perform complex search queries on your data to find exactly what you're looking for.
  • Data Visualization: Use OpenSearch Dashboards, a powerful visualization tool, to create intuitive and interactive dashboards for your data.
  • Security and Alerting: Ensure your data is secure with advanced security features, and set up alerts to stay informed about critical events.

Quick Installation - OpenSearch


To quickly get started using OpenSearch and OpenSearch Dashboards, we will be using docker to deploy opensearch containers.

Before proceeding, you need to install Docker and Docker Compose on your local machine.

Docker Installation steps > Docker Installation Steps


What is Docker Compose ?

Docker Compose is a tool that simplifies the process of defining, running, and managing multi-container applications in Docker. It allows you to configure and orchestrate multiple Docker containers that work together as a single service, using a simple YAML file for configuration.

To install the Opensearch, we need a special file, called a Compose file, that Docker Compose uses to define and create the containers in your cluster.


The OpenSearch Project provides a sample Compose file that you can use to get started. We will download it in below section.


Run Pre requisite commands


Step 1: Run Pre-requisite Commands

  1. Disable memory paging and swapping performance on the host to improve performance: 
    sudo swapoff -a
  2. Edit the sysctl config file that defines the hostâ€™s max map count: 
    sudo vi /etc/sysctl.conf
  3. Set max map count to the recommended value of 262144: 
    vm.max_map_count=262144
  4. Reload the kernel parameters: 
    sudo sysctl -p


For more information link: OpenSearch Docker Installation Guide


Install and Configure OpenSearch with Docker


Step 2: Download the Sample Compose File


Download the sample Compose file to your host. You can use command line utilities like curl and wget, or manually copy docker-compose.yml from the OpenSearch Project documentation repository using a web browser.


To use cURL, send the following request:

curl -O https://raw.githubusercontent.com/opensearch-project/documentation-website/2.18/assets/examples/docker-compose.yml


Setting a Custom Admin Password

Create an empty file named .env and add the following line, then save it:

OPENSEARCH_INITIAL_ADMIN_PASSWORD = <custom-admin-password>

For OpenSearch 2.12 or greater, set a new custom admin password before installation using the following command:

Note: The password must meet the following requirements:

  • Minimum of 8 characters
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one digit
  • At least one special character

Note: Place both the docker-compose.yml and .env files in the same location where Docker is installed.


Run the following command on the command line to set the custom admin password and move to Step 3:

export OPENSEARCH_INITIAL_ADMIN_PASSWORD=<custom-admin-password>


Step 3: Download and Run OpenSearch Container


Run OpenSearch in a Docker container.

Official OpenSearch images are hosted on Docker Hub. If you want to inspect the images, you can pull them individually using docker pull, such as in the following examples:


Docker Hub Commands:


docker pull opensearchproject/opensearch:2
docker pull opensearchproject/opensearch-dashboards:2

Verify Docker Setup with OpenSearch


Before continuing, you should verify that Docker is working correctly by deploying OpenSearch in a single container.


Run the Following Command

This command maps ports 9200 and 9600, sets the discovery type to single-node, and requests the newest image of OpenSearch:


docker run -d -p 9200:9200 -p 9600:9600 -e "discovery.type=single-node" opensearchproject/opensearch:latest

Send a Request to Port 9200

The default username is admin.

For example, if the new password is Learn@123:


curl https://localhost:9200 -ku admin:Learn@123

Expected Response

You should get a response that looks like this:

{
    "name": "opensearch-node1",
    "cluster_name": "opensearch-cluster",
    "cluster_uuid": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "version": {
        "number": "2.x.x",
        "build_flavor": "default",
        "build_type": "docker",
        "build_hash": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
        "build_date": "YYYY-MM-DDTHH:mm:ssZ",
        "build_snapshot": false,
        "lucene_version": "x.x.x",
        "minimum_wire_compatibility_version": "x.x.x",
        "minimum_index_compatibility_version": "x.x.x"
    },
    "tagline": "The OpenSearch Project: https://opensearch.org/"
}

Stop a Running Container (Not in cluster)


Before stopping the running container, display a list of all running containers and copy the container ID for the OpenSearch node you are testing.


docker container ls

Stop the running container by passing the container ID to docker stop:


docker stop <containerId>

Deploy an OpenSearch Cluster Using Docker Compose


Although it is technically possible to build an OpenSearch cluster by creating containers one command at a time, it is far easier to define your environment in a YAML file and let Docker Compose manage the cluster. The following section contains example YAML files that you can use to launch a predefined cluster with OpenSearch and OpenSearch Dashboards.

Steps to Deploy:

  1. Navigate to the Docker directory and run the following command:
    2. docker-compose up -d
  2. Verify that the service containers started correctly:
    3. docker-compose ps
  3. If a container failed to start, review the service logs:
    4. # If you don't pass a service name, docker-compose will show you logs from all of the nodes
docker-compose logs <serviceName>
  4. Open the following URL to start exploring OpenSearch:

    5. http://localhost:5601/app/login

  5. Stop the running containers in your cluster when you're done:
    6. docker-compose down
